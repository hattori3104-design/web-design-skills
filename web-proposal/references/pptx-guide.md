# 受注前提案の .pptx 生成ガイド

タイプA（受注前提案）でスライドを作るときの方針。**必ず `/mnt/skills/public/pptx/SKILL.md` を先に読む**（色・フォント・レイアウト・QAの詳細はそちら）。ここはウェブ制作提案に特化した補足。

---

## 基本方針

- pptxgenjs で生成（テンプレートがなければ）
- 1スライド1メッセージ。文書のように詰め込まない
- 各スライドに視覚要素（図・アイコン・大きな数字・カード）を入れる。テキストだけのスライドにしない
- 左揃え基調（タイトルのみ中央可）

## カラー

ウェブ制作提案は「信頼感・専門性」を出す配色が合う。BtoB（企業向け）なら落ち着いたネイビー・チャコール系、BtoC（個人向け）なら明るく親しみやすいトーンを基本に。pptxスキルのパレットから選ぶか、クライアントのブランドカラーを使う。デフォルトの汎用ブルーは避ける。

- 落ち着いた信頼系：Midnight Executive（navy `1E2761` / ice `CADCFC`）、Charcoal Minimal
- 技術・製造系：Ocean Gradient、Teal Trust
- 表紙・結びは濃い背景、本文は明るい背景の「サンドイッチ」構成にすると締まる

## フォント

日本語スライドは、PowerPoint側でのレンダリングを前提に**游ゴシック（Yu Gothic）またはメイリオ**を指定するのが無難。QAはLibreOffice代替フォントになるため、文字あふれの確認は余裕（スラック）を見て判断する。

| 要素 | サイズ目安 |
|------|-----------|
| スライドタイトル | 36〜44pt 太字 |
| セクション見出し | 20〜24pt 太字 |
| 本文 | 14〜16pt |
| 補足・出典 | 10〜12pt |

---

## ウェブ制作提案でよく使うスライド表現

| 内容 | 表現 |
|------|------|
| 目標数値（月◯件など） | 大きな数字callout（60〜72pt）＋小さなラベル |
| 現状→目標 | before/after の比較2カラム |
| ファネル | 認知→興味→検討→行動の横並びプロセス図 |
| サイトマップ | 階層のツリー図、またはカードのグリッド |
| ポジショニング | 2軸マップ（散布図的に配置） |
| スケジュール | 横タイムライン or フェーズのステップ |
| デザイン方向性 | 参考イメージを面で見せる（ハーフブリード画像＋オーバーレイ） |

---

## やってはいけないこと（pptxスキルより重要分を再掲）

- タイトル下のアクセント線、装飾カラーバー・サイドストライプは入れない（AI生成っぽさが出る）
- 全スライド同じレイアウトの繰り返しを避ける
- 本文の中央揃えを避ける
- 文字あふれを残さない

---

## 生成後QA（必須）

1. `extract-text output.pptx` で内容・誤字・順序を確認
2. PDF化→画像化して**目視で視覚QA**（あふれ・重なり・余白・コントラスト）
3. 見つけた実害のある問題だけ直す。微調整のループはしない

```bash
python /mnt/skills/public/pptx/scripts/office/soffice.py --headless --convert-to pdf output.pptx
rm -f slide-*.jpg
pdftoppm -jpeg -r 150 output.pdf slide
ls -1 "$PWD"/slide-*.jpg
```
