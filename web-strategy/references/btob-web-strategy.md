# BtoB Web戦略 ドメイン知識ベース

BtoB向け（企業が企業に売るビジネス）のWebサイト・集客戦略を扱うときに参照する専門知識。
`web-strategy` のヒアリングで **BtoB（または両方）** と判明した場合、戦略立案の前にこのファイルを読み込み、ここの知見を回答へ具体的に織り込むこと。一般論で答えず、下記の知見（戦略・実行・失敗例）に基づいて、具体的な数字・進め方・対策を盛り込む。

---

## 書き方のルール：専門用語は噛み砕く（ただし、くどくしない）

読み手は、デザイナー本人と、その先のクライアント（企業の経営者・リーダー）。経験ある社会人なので、**説明しすぎない**こと。冗長さは信頼を損なう。

- **基本は素の言葉で書く。** 説明が要らない平易な語を最初から選べば、注釈そのものが不要になる。
- 言い換えを添えるのは、**経営者でも通じにくい“業界の隠語・略語”に限る**（例：EFO、MQL、301リダイレクト）。「見込み客」「事例」「問い合わせ」など通じる言葉は**そのまま使い、括弧での補足を付けない**。
- 言い換えは初出で一度だけ、短く。**括弧を重ねたり、一文に2つ以上の注釈を入れない。**
- 「規模で変わるので本来は確認…」のような前置き・保険は、必要なとき1回だけ。毎回繰り返さない。
- 略語は、開いた方が伝わるなら略語自体を使わず最初から日本語で書く。

## 回答前の内部思考（必ず先に行う）

出力を始める前に、頭の中で次を済ませる。考えた過程は書かず、結論だけを構造化して出す。

1. **規模判定** … この案件はどの規模帯か（下記「案件規模の判定」）。話の深さをそこに合わせる。
2. **戦略選定** … 「この知識ベースに照らすと、この案件で一番効く打ち手は何か」を考える。
3. **失敗例チェック** … 依頼や前提が下記「失敗例」のどれかに当てはまらないか確認する。当てはまれば、望まれた方法でもまずリスクを伝え、代わりの案を出す。
4. **やりすぎチェック** … 小さな案件に、大企業向けの重たい仕組み（自動メール配信、十数万字の大型コンテンツ、専任の営業対応など）を提案していないか見直す。

## 案件規模の判定（やりすぎを防ぐ）

この知識ベースは、もともと中〜大規模のBtoB企業を想定して書かれている。小〜中規模案件が中心の場合は、**規模に応じて深さを必ず加減する**。迷ったら予算感と運用体制（社内で更新できる人がいるか等）をヒアリングする。

| 規模帯 | 目安 | どこまで適用するか |
|--------|------|------------------|
| **小規模** | 〜10万円規模／数ページのサイト・1枚もの（LP）／経営者ひとり〜少人数で運用／Web担当者なし | 目標から逆算する計算は「目標の契約数 → 必要な問い合わせ数」までの簡単なものに留める。自動メール配信や大型コンテンツ、専任の営業対応は**最初の提案に詰め込まない**（将来の選択肢として軽く触れる程度）。まずは基本に集中：信頼を生む実績・事例、わかりやすい申し込みボタン、入力しやすいフォーム |
| **中規模** | 数十万円〜／複数ページ＋事例・ブログ／専任または兼任の担当者あり | 顧客像を3層で整理し（後述）、逆算の計算もきちんと組む。集客の各段階に合わせたコンテンツを設計。自動メール配信は費用対効果を添えて提案 |
| **大規模** | 制作も運用も予算が潤沢／マーケ部門・営業チームあり | 知識ベースをフル適用（しっかりした逆算設計、大型コンテンツ、自動メール配信と顧客管理システムの連携、問い合わせへの即時対応、流入経路ごとの採算管理） |

本ファイルの数字（例：検討期間6〜12ヶ月、関わる人が数名、フォーム改善で問い合わせが約1.7倍、設計を省くと開発費が倍になった等）は、**元資料（NotebookLMから抽出したもの）が言っていること**で、こちらで裏取りした事実ではない。クライアント提案で使うときは、言い切らず「一般にBtoBでは〜という傾向がある」「〜という例が報告されている」と添え、自社案件の実測値とは区別する。

## 1. 戦略と論理

- **BtoBの買い方のクセを押さえる**：企業同士の取引は、(1)決めるまでが長い（一般に半年〜1年と言われる）、(2)関わる人が複数いる、(3)感情より損得・合理性で判断する、という特徴がある。だから「なんとなく良さそう」ではなく、**信頼できること・費用に見合うことを“証拠”で示す**のが効く。
- **顧客を3つの立場で考える**：ターゲットをひとくくりにせず、(1)会社そのもの、(2)担当する部署、(3)実際に動く個人、の3つの目線で捉える。同じ会社でも「最終的に決める人」「選ぶ担当者」「実際に使う人」では気にする点が違うので、それぞれに刺さる材料を用意する。
- **「選んで失敗したくない」という不安を消す**：買い手が一番怖いのは「選定をミスること」。自社ならではの考え方や、深い専門情報をきちんと載せておくと、専門性と信頼の証明になり、競合と差がつく。（※十数万字級の大型コンテンツは大企業の話。小規模なら“中身の濃い記事を数本”で十分代わりになる）
- **目標から逆算して数字を組む**：「とりあえず月◯件」と勘で決めず、**最終ゴール（契約数）から逆算**して、必要な問い合わせ数・訪問数まで分解する。各段階の目標が一本の線でつながるので、何が足りないか一目でわかる。
- **段階ごとに気持ちの変化を設計する**：見込み客は「知る → 興味を持つ → 比べる → 決める」と進む。各段階で“次に進むには何を感じてもらう必要があるか”を読み、それを後押しする情報と次の一手（申し込みボタンなど）を置く。

---

## 2. 進め方と具体策

- **作り始める前に「設計」を終える**：いきなり制作に入らず、まず方向性のすり合わせ → 市場と競合の整理 → 顧客像づくり → 目標の逆算 → ページの骨組み図（ワイヤーフレーム＝デザイン前のラフ設計図）の確定、までを先に済ませる。
- **必要な問い合わせ数の出し方**：最終目標から、ざっくり次のように逆算する。

  ```
  必要な見込み客の数 ＝ 目標の契約数 ÷ (各段階で次に進む割合をかけ合わせたもの)
  ```

  - 小規模案件では、ここまで細かくせず「目標の契約数 ÷ 問い合わせからの成約率 ＝ 必要な問い合わせ数」くらいの簡単な計算で十分。
- **フォームを入力しやすくして取りこぼしを減らす**：入力欄を「名前・会社名・メール・内容」など4〜6項目に絞る。スマホで押す部分は指でタップしやすい大きさ（最低44ピクセル四方）にする。これだけで問い合わせ数が伸びた例が報告されている。
- **熱いうちに連絡する**：資料請求やセミナー直後、相手の関心が高い数時間以内に連絡すると反応が良い。大きな案件では、専用ツールを連携させて、相手の動きに合わせたメールを自動で送る仕組みも使う（※中〜大規模向け。小規模なら手動で“その日のうちに”返すだけでも効果的）。
- **使えるツール**：

  | 用途 | ツール |
  |------|--------|
  | 図や設計の作成 | Miro, Whimsical |
  | アクセス分析 | GA4（どこで離脱しているか見る）, PageSpeed Insights（表示が3秒以内か確認） |
  | 進行管理 | Backlog, Asana（誰がいつまでに何をやるかを見える化） |

## 3. やりがちな失敗と対策

依頼や前提が以下に当てはまるときは、**たとえ相手がその方法を望んでいても、まずリスクを伝えてから対策・代替案を添える**。

| 失敗 | 何が起きるか | 対策・代わりの案 |
|------|------------|----------------|
| **目先だけ最適化する** | 「1件あたりの獲得コスト」を全経路一律で課すと、リピートの少ない経路に予算が偏り、集めるほど赤字になる | 経路ごとに「その客が結局いくら使ってくれるか」を毎月見直し、獲得コストの上限を経路別に調整 |
| **設計を飛ばして作り始める** | 仕様を固めずに制作着手 → 後から変更が続出し、費用が当初見積もりの倍になった例も | 「どう売上を作るか」と「必要な機能」を紙1枚で先に合意し、サインをもらう |
| **終盤でのちゃぶ台返し** | 完成間際に社長など決裁者が「やっぱり違う」と覆す | 早い段階から決裁者を確認してレビューに巻き込み、決まったことを記録に残す |
| **公開後の管理を軽視** | 保守契約を結ばず放置 → 証明書の更新漏れやシステムの穴を突かれて改ざん被害 | 最初の予算に月々の管理費（月額数万〜）を入れ、バックアップを自動化 |
| **リニューアルで検索評価を失う** | 古いページから新しいページへの転送設定を忘れ、これまで積み上げた検索順位がリセットされる | 旧URLと新URLの対応表を作り、公開前チェックで「検索エンジンに載せない設定の解除」を必須項目にする |

---

## まとめ（核となる考え方）

> BtoBで成果を出す鍵は、「目標の契約数から逆算した数字の設計」と「相手の不安を消す情報の見せ方」を、**作り始める前のすり合わせ段階でやり切ること**にある。

ただし小規模案件では、これを**規模に合わせて翻訳**する。小規模なら「簡単な逆算 ＋ 信頼を示す最低限の情報 ＋ 着手前の合意」に圧縮し、やりすぎを避けることが、むしろ成果と満足につながる。
