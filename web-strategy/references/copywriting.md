# コピーライティング ドメイン知識ベース

Webサイト・LP・広告のキャッチコピーやボディコピーを書く・レビューするときに参照する専門知識。
次のような依頼が来たら、このファイルを読んで知見を回答に織り込む。一般論で埋めず、下記の論理・具体策・失敗例を使う。

- 「キャッチコピーを考えて」「言葉が弱い気がする」「響かない」
- 「ファーストビューの文言を直して」「CTAの文言どうする？」
- 「このコピー、クライアントにどう説明する？」
- 「ボディコピーの構成を整えて」「訴求の切り口を変えたい」

## 書き方のルール（他の参照ファイルと共通）

読み手はデザイナー本人と、その先のクライアント（経営者・リーダー）。**説明しすぎない**。

- 通じにくい隠語・略語（VoC、PASONA、CTA など）だけ、初出で一度だけ短く言い換える。
- 本ファイルの数値（5秒の法則、60,000倍、CVR2倍など）は**元資料の主張**で裏取りした事実ではない。クライアント提案では「〜という傾向が報告されている」と添え、断定しない。

## 1. 戦略と論理

### コンバージョンを決める要素の優先順位

成果に影響する要素は、重要度の順に次のとおり。

```
影響大 → 動機（ターゲットの悩みに刺さるか）
       → 価値の伝わり方（何が得なのかが明確か）
       → 心理的障壁を下げる要素（信頼・実績・リスク軽減）
影響小 → ボタンの色・細部のデザイン
```

- **改善の順番**：サイト全体の導線 → 言葉と構成 → 信頼要素 → ボタン色などの細部。言葉が整う前にデザインを磨いても、「抜けたバケツに水を注ぐ」行為になる。
- ユーザーはページに着いてから**5秒以内**に「自分に関係があるか」を判断し、留まるか離脱するかを決める（元資料の主張）。脳が視覚情報を先に処理するため、最初の印象は文章より画像が先に届く。

### ベネフィットの3段階：どの深さで訴求するか

```
スペック（仕様）  → 「防水性能IPX7」
メリット（利点）  → 「水に濡れても壊れない」
ベネフィット（価値体験） → 「雨の日でも地図を気にせず走れる」
             さらに深く → 「完走できた自分を誇れる」（情緒的・自己表現的価値）
```

- 顧客が買うのは「ドリル」ではなく「穴」、さらにその先の「棚を作って部屋がすっきりした生活」。訴求はできる限りベネフィット層まで深く掘る。
- ただし深掘りしすぎて「わかりにくい詩」になるのは逆効果（→ 失敗例の「抽象的ポエム」参照）。

### 社会的証明：他人の選択で不安を消す

- 「満足度98%」「導入実績1,200社」「★4.8（302件）」のような具体的数値は、ひとりで判断することへの不安を下げる。数字のない「大好評」は逆に信頼を下げる。
- ネガティブなレビューを全部消すと、かえって「都合の良い情報だけを選んでいる」と判断されて信頼が落ちる（→ 失敗例参照）。

## 2. 実行手順と具体策

### ステップ0：コピーを書く前に「顧客の言葉」を集める（VoCリサーチ）

コピーを書く前に、**自分の言葉ではなく顧客が使っている言葉**を集める。これをVoC（Voice of Customer＝顧客の声）リサーチという。

収集先（4ソース）：
1. 自社のレビュー・お問い合わせ履歴
2. Amazonや楽天の競合商品レビュー
3. Reddit・発言小町・知恵袋などのQ&Aサービス
4. Facebookグループ・SNSの投稿

集める量の目安：200〜400件（元資料の主張）。要約せず「生の言葉のまま」抜き出す。

- **悪い例**：「関節の違和感が気になる」（デザイナーが意訳した言葉）
- **良い例**：「孫を抱っこするのが怖い」（顧客が実際に使った言葉）

頻出するリアルな言葉を、そのままキャッチコピーや見出しに使う。

### ファーストビューの5要素

| 要素 | 目安 | ポイント |
|------|------|---------|
| キャッチコピー | 15文字以内 | 3秒で価値が伝わる。具体的な数字を入れる |
| サブコピー | 30〜50文字 | キャッチで言い切れなかったベネフィットを補強 |
| メインビジュアル | — | 商品・サービス利用後の「理想の未来」を見せる |
| CTA（申し込みボタン） | — | 「詳しくはこちら」ではなく「無料で資料をダウンロード」など具体的な行動を書く |
| 信頼要素 | 2〜3つに絞る | 実績数字・受賞・メディア掲載ロゴなど |

### 文章の「型」の使い分け

**PASONA（感情的共感が先に必要な場合）**
```
Problem（問題提起）→ Affinity（共感・親近感）→ Solution（解決策）
→ Offer（提案）→ Narrowing down（対象を絞る）→ Action（行動）
```
- BtoC・生活系・悩み訴求に向く。共感で「自分のことだ」と感じさせるのが肝。

**QUEST（専門性・説明が必要な場合）**
```
Qualify（誰向けかを絞る）→ Understand（悩みを理解している）→ Educate（知識を伝える）
→ Stimulate（欲求を刺激する）→ Transition（行動へ移す）
```
- 高額商品・専門サービス・BtoBに向く。「まず教育」が必要な案件で使う。

### マイクロコピーで「最後の一歩」を踏ませる

CTAボタンの直下に小さな一言を添えるだけで、申し込みへの心理的なハードルが下がる。

- 「※キャンセルはいつでも可能」
- 「1分で完了」
- 「クレジットカード不要」
- 「まず見るだけでも大丈夫です」

元資料ではCVRが約2倍になった例が報告されている（数字は主張として扱う）。

### ABテストで科学的に磨く

- **1テスト＝1変更だけ**：文言・画像・色を同時に変えると、何が効いたか判定できない。
- **十分な期間と数**：最低1週間〜1ヶ月。成約数が各パターン50件以上になるまで判断しない（元資料の主張。少ない数での判断は統計的に信頼できない）。
- **ヒートマップを使う**：Microsoft Clarity（無料）などで「どこまで読まれているか」「どこで読むのをやめているか」を見る。誰も読んでいないコンテンツは削除か後半に移す。

## 3. 失敗例と対策

依頼の内容が以下に当てはまるときは、**たとえ相手がその方法を望んでいても、まずリスクを伝えてから対策・代わりの案を添える**。

| 失敗 | 何が起きるか | 対策・代わりの案 |
|------|------------|----------------|
| **デザインを先に磨く** | 言葉と構成が弱いままデザインを仕上げると、見た目は良くても成約が増えない | まず文章の主語を「私たちは〜」から「あなたは〜」に変えることから始める。言葉を先に固めてからデザインに入る |
| **抽象的ポエム訴求** | 「未来を創造する」「笑顔あふれる社会へ」などは、ユーザーも検索エンジンもスルーする | 「ターゲットがその言葉を自分の口で叫びたくなるか」で検証する。例：「毎日の帳簿処理を楽にしたい！」なら刺さる |
| **複数要素を一度にテスト** | 文言・画像・色を同時に変えると、何が効いたか特定できず、次に生かせない | 1テスト＝1変更だけ。仮説を立てて1つずつ検証する |
| **広告とLPの言葉がズレる** | 広告で「無料体験」と言ってLPに誘導したのに、LPに「無料体験」の文字が見当たらないと「騙された」と感じて即離脱 | 広告の言葉・LPのキャッチ・本文を一本のストーリーでつなげる。広告で約束したことは必ずFVで受け取る |
| **ネガティブな口コミを全削除** | 都合の良い情報だけを載せると、かえって「隠している」と思われ信頼が下がる | ネガティブな声にも公式の補足コメントを添えて中和する。完璧すぎるより少しの傷がある方が信頼されやすい |
