# HTMLワイヤーフレーム 生成ガイド

モードAでHTMLワイヤーフレームを作るときの詳細仕様。
CSSトークン・コンポーネント・アノテーションの書き方を定義する。

---

## カラートークン（グレースケール固定）

```css
:root {
  --wf-bg:       #F7F7F7;  /* ページ背景 */
  --wf-surface:  #FFFFFF;  /* セクション白背景 */
  --wf-gray:     #F0F0F0;  /* セクション薄グレー背景 */
  --wf-block:    #E2E2E2;  /* プレースホルダーブロック */
  --wf-img:      #D5D5D5;  /* 画像プレースホルダー */
  --wf-border:   #CCCCCC;  /* 罫線・ボーダー */
  --wf-dark:     #555555;  /* 本文テキスト */
  --wf-darker:   #222222;  /* 見出しテキスト */
  --wf-muted:    #AAAAAA;  /* ラベル・キャプション */
  --wf-cta:      #444444;  /* CTAボタン */

  /* アノテーション */
  --wf-note-bg:  #FFFBE6;
  --wf-note-bdr: #E0C84A;
  --wf-note-txt: #7A6500;

  /* レイアウト */
  --max-w: 1120px;
  --side:  24px;
}
```

**絶対に使わないこと**：ブランドカラー・アクセントカラー・グラデーション。デザイン工程との混同を避ける。

## 必須コンポーネント

### 管理バー（ページ最上部）
```html
<div class="wf-page-title">
  <strong>WF: [クライアント名] / [ページ名].html</strong>
  <span>対象デバイス: PC（1120px）</span>
  <span>ファネル: [このページが担うファネル段階]</span>
  <span>主CTA: [主要なCTA]</span>
</div>
```

### アノテーションボックス
各セクションの直後に配置する。必ず以下の情報を含める：
- **セクション名 ／ ファネル段階**
- 素材の出所（支給 / 自社で用意 / 確認が必要）
- クライアントへの確認事項（情報収集が必要な項目）
- レイアウト・実装上の注意

```html
<div class="wf-note">
  <strong>SECTION名 ／ ファネル：認知〜興味</strong>
  [説明テキスト]
</div>
```

### 画像プレースホルダー
```html
<div class="wf-block wf-img-placeholder" style="height: 340px;">
  [写真の説明]（支給 / 撮影が必要）
</div>
```

### CTAボタン
```html
<!-- プライマリCTA -->
<div class="wf-btn">ボタンラベル</div>
<!-- セカンダリCTA（アウトライン） -->
<div class="wf-btn wf-btn--sub">ボタンラベル</div>
```

### テーブル・グリッド
```html
<div class="wf-grid-3">  <!-- 3カラムグリッド -->
<div class="wf-grid-2">  <!-- 2カラムグリッド -->
<div class="wf-grid-4">  <!-- 4カラムグリッド -->
```

## セクションの背景色パターン

視覚的な区切りをグレースケールで表現する。

| セクション | 背景 | 用途 |
|-----------|------|------|
| `wf-section` | `#FFFFFF` | 通常セクション |
| `wf-section wf-section--gray` | `#F0F0F0` | 交互配置・強調 |
| Hero / CTA帯 | `#2C2C2C`〜`#333333` | ダークセクション（`color:#FFF`を忘れず） |

白→グレー→白→グレーで交互に配置するのが基本。HeroとメインCTA帯はダーク背景で独立させる。

## ページの縦サイズ目安

| セクション | 縦サイズ目安 |
|-----------|------------|
| HEADER | 64px（sticky） |
| HERO | 520〜600px |
| 3カラムカード | 自動（コンテンツ次第） |
| stat strip（数字帯） | 120〜160px |
| 2カラムコンテンツ | 400〜480px |
| FAQセクション | 自動 |
| メインCTA帯 | 200〜280px |
| FOOTER | 240〜320px |

## アノテーションで必ず書くこと

| 項目 | 書き方の例 |
|------|----------|
| ファネル段階 | `ファネル：認知〜興味` |
| 素材の出所 | `写真は工場全景（支給素材から選定）` |
| 確認事項 | `最小ロット数・対応板厚はクライアントへの確認が必要` |
| 実装メモ | `モバイルはハンバーガーメニューに切り替え` |
| 将来対応 | `将来的にカルーセル化を検討` |

## よく使う日本語コピーのひな形

ワイヤー段階では実際のコピーに近いテキストを入れる（「テキストテキスト」は使わない）。
web-strategyで出たキャッチコピー・強みの言葉をそのまま使う。

```
HERO見出し例：「[強みA]・[強みB]に強い／[サービス]」
CTAボタン例：「お問い合わせ・図面送付」「取引実績を見る →」
数字例：「45+年」「5社」「最短3日」
バッジ例：「ISO9001 認証」「全品検査対応」
FAQのQ例：「最小ロットは何個から？」「見積もりはいつ返答？」
```
